AI Inference: What happens after the model is trained?

To know where to start optimising an LLM, we first need to understand what happens when it turns an input into an output.

Cue / visual anchorMain notes / key thinking
Overview / Chapter 01

Two stages of inference

Inference begins after training. It is the process of using a trained model to respond to an input.

The inference story

This write-up grew out of optimising Qwen3.5 for the AI-HPC26 competition. Before deciding where to optimise, I needed a clear picture of what actually happens inside an LLM during inference.

At a high level, inference has two parts:

  1. PrefillThe model processes all of the input tokens and builds the context it will use to answer.
  2. DecodingThe model generates the output one token at a time.

This chapter begins at the entrance to prefill: the tokenizer.

Tokenizers turn text into model-sized pieces

Start with the familiar prompt: “The cat sat on the”. A language model does not receive this as one continuous sentence. The tokenizer first breaks it into discrete units called tokens.

Different tokenizers may split the same text differently:

  • Phrase-like: {“The cat”, “sat”, “on the”}
  • Word-like: {“The”, “cat”, “sat”, “on”, “the”}
  • Subword-like: {“Th”, “e”, “cat”, “sat”, “on”, “th”, “e”}

Modern production tokenizers commonly use subword or byte-level pieces. This gives a model a practical vocabulary: it can reuse frequent chunks while still representing unfamiliar words.

Why it matters

Text is not always plain words

Emoji, symbols, code, and multilingual text all need a route into tokens.

🙂→UTF-8 bytes→token IDs

A tokenizer defines what the model sees

Thinking in phrases, words, or parts gives us different ways to learn patterns in language. Subword and byte-level approaches also give the tokenizer a fallback for special characters, emoji, symbols, and text from many languages.

The text is encoded, segmented, and mapped to numerical token IDs. An emoji may become one token or several, depending on the tokenizer. Those token IDs—not the visual characters themselves—are what the model processes during prefill.

Before we optimise the model’s work, we need to understand the units of work.
Summary / take this with you

Inference has two stages: prefill processes the input, and decoding generates the output. Tokenization is the first step—it determines the units the model actually sees.
